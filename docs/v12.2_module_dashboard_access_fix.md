# v12.2 구현 보고서 — 모듈 대시보드 노출 권한 추가 수정

**작성일**: 2026-07-21  
**범위**: v12.1에서 놓친, 관리자용 집계 화면인 4개 모듈 대시보드의 employee 노출 문제 수정

---

### 1. 증상

v12.1 반영 이후에도 정성반찬 employee 계정(이수아)의 사이드바에 "HR 대시보드" · "회계 대시보드" · "생산 대시보드" · "유통 대시보드"가 계속 노출됨

---

### 2. 원인

이 4개 항목은 `dashboard/layout.tsx`의 `MENU_ACCESS`에 `viewer: true`로 하드코딩되어 있었고, "권한 관리" 화면의 `DEFAULT_ACCESS`·`LABEL_TO_HREF`에도 아예 포함되어 있지 않아 관리자가 개별적으로 끌 수도 없는 상태였다. v12.1에서는 "권한 관리" 화면에서 실제로 커스터마이즈된 항목(직원 관리 · 영수증 OCR · 경비 정산 · 거래처 관리)만 비교 대상으로 삼아, 애초에 커스터마이즈 대상에 없던 이 4개 항목은 점검 범위에서 빠졌다

각 대시보드(`hr/page.tsx` 등)의 실제 내용을 확인한 결과 재직 인원 · 부서 수 · 신규 입사 등 사업장 전체 집계와 관리자용 바로가기로 구성되어 있어, 일반 직원에게 노출할 근거가 없는 화면으로 판단

---

### 3. 조치

`MENU_ACCESS`에서 4개 모듈 대시보드의 `viewer`를 `false`로 변경 — `/dashboard/integrated`(통합 대시보드)가 이미 관리자 · 매니저 전용으로 하드코딩되어 있던 것과 동일한 방식

| 메뉴 | 이전(viewer) | 수정 후(viewer) |
|---|---|---|
| HR 대시보드(`/dashboard/hr`) | true | false |
| 회계 대시보드(`/dashboard/accounting`) | true | false |
| 생산 대시보드(`/dashboard/production`) | true | false |
| 유통 대시보드(`/dashboard/distribution`) | true | false |

---

### 4. 검증

| 항목 | 결과 |
|---|---|
| `npx tsc --noEmit` | ✅ 오류 없음 |
| 이수아(정성반찬 employee) 사이드바 노출 예상 항목 재계산 | ✅ 인사(계약서 · 휴가관리) · 회계(영수증OCR · 경비정산) · 생산(품목·재고 · 생산지시서 · 재고실사) · 유통(수주관리 · 배송지시) · 알림센터 · AI회계비서 · 환경설정 · 도움말만 남음, 4개 모듈 대시보드 전부 제외 확인 |
| 개발 서버(프론트 3000) 응답 확인 | ✅ 200 |

---

### 5. 참고

사이드바 메뉴 숨김은 UI 레벨 제어이며, 해당 페이지 URL에 직접 접근하는 것 자체를 막는 라우트 가드는 아니다. 이는 v9.x부터 이어진 기존 설계 범위이며, 이번 수정은 그 틀 안에서 노출 여부만 정정한 것이다
