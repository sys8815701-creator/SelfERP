# v12.1 구현 보고서 — 신규 사업장 메뉴 권한 기본값 불일치 수정

**작성일**: 2026-07-21  
**범위**: 사업장별 커스텀 권한 설정이 없을 때 적용되는 사이드바 메뉴 노출 기본값을 실사용 기준에 맞춰 정정

---

### 1. 증상

정성반찬(신규 사업장)의 employee 역할 계정(이수아, `lee.staff@jeongsung.kr`)으로 로그인 시, 행복한 베이커리의 employee 계정(정하늘)과 달리 사이드바에 관리자 · 매니저용 메뉴가 그대로 노출됨. 실제 API 응답의 데이터 접근 범위(급여 · 계좌 등 민감 필드 redaction, 본인 기록 한정 조회)는 두 사업장 모두 정상 동작 — 화면 노출 여부에서만 차이가 발생

---

### 2. 원인

- `GET /api/settings/role-access`는 사업장별로 저장된 커스텀 설정이 있을 때만 값을 반환하고, 없으면 `null`을 반환한다
- 프론트엔드는 이 값이 `null`이면 `dashboard/layout.tsx`의 하드코딩된 기본값 `MENU_ACCESS`로 폴백한다
- 행복한 베이커리는 과거에 관리자가 "권한 관리" 화면에서 직원 관리 · 영수증 OCR · 경비 정산 · 거래처 관리 4개 항목의 뷰어(employee) 권한을 직접 수정해 DB에 저장해두었지만, 이 수정 내용이 코드의 기본값(`MENU_ACCESS`, `access/page.tsx`의 `DEFAULT_ACCESS`)에는 반영되지 않은 상태였다
- 결과적으로 커스텀 설정이 존재하는 행복한 베이커리는 정정된 권한으로 보이고, 커스텀 설정이 아예 없는 신규 사업장은 예전의 느슨한 기본값을 그대로 물려받는다

---

### 3. 조치

`dashboard/layout.tsx`의 `MENU_ACCESS`와 `integrated/access/page.tsx`의 `DEFAULT_ACCESS`를 행복한 베이커리의 실제 저장값과 동일하게 수정 — 커스텀 설정 없이도 모든 신규 사업장이 처음부터 동일한 기준을 적용받는다

| 메뉴 | 이전 기본값(viewer) | 수정 후(viewer) |
|---|---|---|
| 직원 관리(`hr/employees`) | true | false |
| 영수증 OCR(`ocr`) | false | true |
| 경비 정산(`expense`) | false | true |
| 거래처 관리(`accounting/vendors`) | true | false |

직원 관리 · 거래처 관리는 관리 성격이 강해 employee 노출을 막고, 영수증 OCR · 경비 정산은 본인 명의 신청(v11.5에서 이미 백엔드 스코프 확정)이 실무상 가능해야 하므로 노출을 허용하는 방향

---

### 4. 검증

| 항목 | 결과 |
|---|---|
| 이수아(정성반찬 employee)로 `GET /api/hr/employees` · `GET /api/hr/payroll` | ✅ 기존부터 정상(민감필드 redaction, 본인 기록만) — 데이터 스코프는 이번 수정 대상 아님 |

| `npx tsc --noEmit` | ✅ 오류 없음 |
