## 2. 데이터베이스 스키마 검증

### 신규 테이블 (7개)

| 테이블 | 주요 컬럼 | 제약조건 | 상태 |
|--------|----------|---------|------|
| `vendors` | +17 필드 확장 | vendor_type ENUM(매출처/매입처/양방향/기타) | ✅ |
| `account_receivables` | amount, paid_amount, due_date, status | status ENUM(미수/일부수금/완료/대손) | ✅ |
| `account_payables` | amount, paid_amount, due_date, status | status ENUM(미지급/일부지급/완료) | ✅ |
| `tax_invoices` | direction, supply_amount, tax_amount, total_amount | direction ENUM(발행/수취), status ENUM(3종) | ✅ |
| `estimates` | doc_type, supply/tax/total_amount, status | doc_type ENUM(견적서/청구서/발주서), cascade delete | ✅ |
| `estimate_items` | quantity, unit_price, amount | estimate_id FK → cascade | ✅ |
| `budget_items` | budget_year, budget_month, category, btype | 복합 유니크(business/year/month/category/btype) | ✅ |

### 관계 무결성
- `Business` ↔ 회계 7개 테이블: back_populates 양방향 ✅
- `Vendor` ↔ AR, AP, TaxInvoice, Estimate (FK nullable) ✅
- `Estimate` ↔ `EstimateItem`: cascade="all, delete-orphan" ✅

## 3. API 엔드포인트 검증

### 인증·권한
- 모든 회계 엔드포인트: `Authorization: Bearer {token}` + `X-Business-Id` 헤더 필수 ✅
- 타 사업장 데이터 접근 차단 (business_id 필터) ✅

### 비즈니스 로직 검증

| 항목 | 검증 내용 | 결과 |
|------|----------|------|
| 세금계산서 세액 | tax_amount=None 시 공급가액×10% 자동 계산 | ✅ |
| 세금계산서 합계 | total = supply + tax 자동 계산 | ✅ |
| 견적서 품목 합계 | supply = Σ(quantity × unit_price), tax = supply × 0.1 | ✅ |
| 견적서 품목 수정 | PUT 시 기존 아이템 삭제 → 재생성 (cascade) | ✅ |
| 예산 중복 방지 | 동일 연월·항목·구분 중복 등록 시 400 반환 | ✅ |
| AR 연체 필터 | due_date < today AND status NOT IN (완료, 대손) | ✅ |
| AP 연체 필터 | due_date < today AND status != 완료 | ✅ |
| 부가세 납부 예정 | 매출세액 합계 - 매입세액 합계 | ✅ |
| 자금흐름 예측 | 현재 은행잔고 + Σ(AR 수금예정) - Σ(AP 지급예정) | ✅ |
| 대차대조표 검증 | 자산 == 부채 + 자본 check 필드 반환 | ✅ |

### 재무제표 데이터 기반

| 재무제표 | 기반 데이터 |
|---------|-----------|
| 손익계산서 | BankTransaction(입금=매출/출금=비용) + CardSale(카드매출) |
| 대차대조표 | BankTransaction.balance + AR잔액(자산) / AP잔액(부채) |
| 현금흐름표 | BankTransaction 월별 입출금 집계 |
