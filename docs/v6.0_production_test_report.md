## 2. 데이터 모델 검증

### 2.1 생성된 테이블
| 모델 | 테이블명 | 주요 컬럼 |
|------|----------|-----------|
| `Item` | `items` | item_type(ENUM 4종), current_stock, safety_stock, max_stock (Numeric 15,3) |
| `BOM` | `boms` | product_id(FK→items), version, is_active |
| `BOMLine` | `bom_lines` | bom_id(FK), item_id(FK), quantity |
| `ProductionOrder` | `production_orders` | status(ENUM 4종), planned_qty, planned_date |
| `ProductionResult` | `production_results` | order_id(FK), completed_qty, defect_qty |
| `InventoryLog` | `inventory_logs` | log_type(ENUM 6종), quantity(±), log_date |

### 2.2 자동화 동작 확인
- **생산 실적 등록 시**: `ProductionOrder.status` 자동 변경 (생산중/완료)
- **생산 실적 등록 시**: `Item.current_stock` 자동 증가 + `InventoryLog(생산완료)` 자동 생성
- **입출고 등록 시**: `Item.current_stock` 자동 가감
- **재고 실사 적용 시**: 차이 있는 품목만 `InventoryLog(조정)` 생성 후 `current_stock` 업데이트

## 3. API 엔드포인트 검증

### 3.1 `/api/production/*` 전체 목록
| Method | Path | 기능 |
|--------|------|------|
| GET | `/summary` | 대시보드 요약 4종 |
| GET/POST | `/items` | 품목 목록/생성 |
| PUT/DELETE | `/items/{id}` | 품목 수정/삭제 (이력 있으면 삭제 불가) |
| GET/POST | `/boms` | BOM 목록/생성 |
| DELETE | `/boms/{id}` | BOM 삭제 (cascade) |
| GET/POST | `/orders` | 지시서 목록/생성 |
| PUT | `/orders/{id}` | 지시서 수정 (상태 변경 포함) |
| GET/POST | `/results` | 실적 목록/등록 |
| GET/POST | `/inventory-logs` | 이력 조회/등록 |
| GET | `/cost-analysis` | BOM 기반 원가 분석 |
| GET/POST | `/stock-audit` | 재고 실사 준비/적용 |
| GET | `/safety-stock-alerts` | 안전재고 부족 목록 |
| GET | `/efficiency` | 생산 효율 분석 |

### 3.2 수정 사항 (v5.1 후 발견 및 수정)
1. **`GET /results` 미구현** → v5.1 후 즉시 추가
2. **`InventoryLogCreate.reference_no`** 필드명 불일치 (`ref_no` vs `reference_no`) → 별칭 처리로 해결
3. **`log_date` 필수 필드** → Optional 처리, 미입력 시 오늘 날짜 자동 설정
4. **재고 실사 `book_qty` 갱신 타이밍** → `item.current_stock` 조회 전에 이미 변경됨 → 커밋 전 이전 값 캡처로 수정

## 5. 알려진 제한사항

| 항목 | 내용 | 권장 해결책 |
|------|------|------------|
| 인건비 원가 미포함 | 단위 원가 = 자재비만 계산, 노무비·제조경비 미반영 | 품목에 가공비 입력 필드 추가 (v7.x 개선 예정) |
| BOM 버전 관리 | 동일 제품에 여러 버전 BOM 등록 가능하나 기준 BOM 지정 없음 | `is_default` 필드 추가 고려 |
| 재고 실수 입력 | 음수 재고 입력 방지 미구현 | 프론트엔드 `min="0"` 제약 추가 권장 |
| 생산 지시서 삭제 | 현재 DELETE 엔드포인트 미구현 | 취소 상태로 대체 가능 |
| `planned_date` 순서 | NULL인 경우 정렬 순서 미정의 | `NULLS LAST` 정렬 추가 권장 |
