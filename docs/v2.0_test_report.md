## 2. 데이터베이스 스키마 검증

### 신규 테이블 (6개)

| 테이블 | 주요 컬럼 | 제약조건 | 상태 |
|--------|----------|---------|------|
| `departments` | id, business_id, name, code, parent_id | parent_id → self FK | ✅ |
| `positions` | id, business_id, name, level | level: 정수형 | ✅ |
| `employees` | id, business_id, dept_id, pos_id, hire_date | status ENUM(재직/휴직/퇴직) | ✅ |
| `contracts` | id, business_id, employee_id, sign_status | sign_status ENUM(4종) | ✅ |
| `leaves` | id, business_id, employee_id, days(Decimal) | days 0.5 단위 | ✅ |
| `payrolls` | id, business_id, employee_id, pay_year, pay_month | (employee, year, month) 복합 유니크 | ✅ |
| `severances` | id, business_id, employee_id, resign_date | total_pay: Decimal(15,2) | ✅ |

### 관계 무결성
- `Business` ↔ HR 6개 테이블: back_populates 양방향 ✅
- `Employee` ↔ Department, Position, Contract, Leave, Payroll, Severance ✅
- `Department` 자기참조(parent/children) ✅

## 4. 프론트엔드 검증
