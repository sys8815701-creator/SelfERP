# v12.5 구현 보고서 — 소셜 로그인 콜백 인증 헤더 누락 수정

**작성일**: 2026-07-22  
**범위**: 소셜 로그인(카카오 · 네이버 · 구글) 완료 후 `/api/auth/me` 호출이 항상 401로 실패하던 문제 수정

---

### 1. 증상

배포 환경에서 소셜 로그인 진행 시 제공자 인증까지는 정상 완료되지만, 콜백 처리 중 `GET /api/auth/me` 요청이 401로 실패하며 로그인이 완료되지 않음 — 카카오 · 네이버 전부 동일하게 발생

---

### 2. 원인

`frontend/src/app/auth/callback/page.tsx`에서 `/api/auth/me` 호출 시 토큰을 URL 쿼리 파라미터(`?token=...`)로 전달하고 있었음. 백엔드의 `get_current_user`(`core/deps.py`)는 `Authorization: Bearer <token>` 헤더만 확인하므로, 쿼리 파라미터로 전달된 토큰은 애초에 읽히지 않아 매 요청이 401로 처리됨

같은 파일의 사업장 조회 호출(`GET /api/business/`)은 정확히 헤더로 토큰을 전달하고 있어 정상 동작 — `/api/auth/me` 호출 한 곳만 이 처리가 누락되어 있었음. 로컬 환경에서도 동일하게 실패했을 코드지만, 이번에 배포 후 소셜 로그인을 처음부터 끝까지 실제로 테스트하면서 발견됨

---

### 3. 조치

`/api/auth/me` 호출에 `Authorization: Bearer <token>` 헤더를 추가해 사업장 조회 호출과 동일한 방식으로 통일
