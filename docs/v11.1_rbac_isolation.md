# v11.1 구현 보고서 — 사업장별 역할(RBAC) 분리

**작성일**: 2026-07-21  
**범위**: 인가 구조를 계정 전역 role에서 사업장별 role로 전환

---

### 1. 스키마 변경

| 테이블 | 변경 | 비고 |
|---|---|---|
| `employees` | `role` 컬럼 추가 (Enum: admin · accountant · employee, 기본값 employee) | 사업장 소속 기준 역할 |
| `users` | `is_platform_admin` 컬럼 추가 (Boolean, 기본값 false) | 사업장 소속과 무관한 플랫폼 운영자 플래그 |

기존 `Employee.user_id`가 연결된 행은 서버 기동 시 1회 한정으로 연결된 `User.role` 값을 `Employee.role`에 백필 — 기존 3개 계정(행복한 베이커리)의 실제 권한은 변화 없음

---

### 2. 신규 인가 함수 (`core/deps.py`)

| 함수 | 반환 | 판정 순서 |
|---|---|---|
| `get_current_role` | 현재 `X-Business-Id` 사업장에서의 역할 | 소유자면 admin → 재직 중인 Employee.role → 매칭 없으면 employee |
| `get_role_for_business` | 위와 동일하나 `business_id` 경로 파라미터를 쓰는 라우터(`business.py`)에서 직접 호출하는 순수 함수 버전 |

---

### 3. 호출부 이관

`current_user.role` 기반 인가 체크를 `get_current_role`(또는 `get_role_for_business`) 기반으로 전환한 파일

| 파일 | 대상 |
|---|---|
| `business.py` | `update_business` · `update_bank_info` · `subscribe_pro` · `unsubscribe_pro` · `approve_join_request` · `reject_join_request` |
| `distribution.py` | `require_writer` |
| `expense.py` | 목록 조회 2건 · `approve_expense` · `reject_expense` |
| `hr.py` | `require_manager` |
| `ledger.py` | 계정과목 생성 · 수정 · 삭제 |
| `payroll.py` | `require_admin` (admin 전용 → admin · accountant로 정상화) |
| `production.py` | `cost-analysis` |
| `settings.py` | `role-access` 저장 |
| `statements.py` | `require_admin` |

---

### 4. 플랫폼 관리자 전용으로 이관

사업장 소속과 무관한 작업이므로 `role=="admin"` 체크를 `is_platform_admin` 체크로 교체

- `auth.py`: `GET /users/pending` · `PATCH /users/{id}/approve` · `DELETE /users/{id}`
- `pending_registration.py`: `GET /list` · `POST /{id}/review`

---

### 5. 사업장 스코프로 재작성

| 엔드포인트 | 이전 | 이후 |
|---|---|---|
| `GET /api/auth/users` | 소유한 모든 사업장 구성원(`_my_org_user_ids`) | `X-Business-Id` 사업장의 재직 중인 Employee만, role은 해당 사업장 기준으로 응답 |
| `PATCH /api/auth/users/{id}/role` | 전역 `User.role` 수정 | 해당 사업장 `Employee.role`만 수정, 대상자의 다른 사업장 권한에는 영향 없음 |

`_my_org_user_ids` 헬퍼는 더 이상 쓰이지 않아 제거

### 6. 검증

| 항목 | 결과 |
|---|---|
| 백엔드 `python -c "import main"` | ✅ |
| 프론트 `npx tsc --noEmit` | ✅ |
| 서버 기동 시 컬럼 자동 추가 + 백필 | ✅ (`employees.role`, `users.is_platform_admin` 확인) |
| `GET /api/auth/users` (사업장 4, 기존 3계정) | ✅ admin · accountant · employee 정상 응답 |
| `PATCH /api/auth/users/18/role` → accountant | ✅ `Employee.role`만 변경, `User.role`(전역)은 employee로 유지 확인 |
| 사업장 admin 계정으로 `GET /api/auth/users/pending` 호출 | ✅ 403 (플랫폼 관리자 아님) |
| 사업장 admin 계정으로 `GET /api/pending-registration/list` 호출 | ✅ 403 (플랫폼 관리자 아님) |

---

### 7. 알려진 제약

`integrated/pending/page.tsx`의 "개인 계정 승인" · "사업자 등록 승인" 탭은 이제 플랫폼 관리자만 호출 가능한 API를 사용하므로, 사업장 admin이 이 페이지에 접근하면 해당 두 탭에서 403이 발생한다. v11.2에서 개발자 콘솔을 신설해 두 탭을 이관하고 이 페이지에서는 제거할 예정
