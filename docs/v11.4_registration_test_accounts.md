# v11.4 구현 보고서 — 가입 승인 흐름 테스트 계정

**작성일**: 2026-07-21  
**범위**: 3가지 가입 승인 플로우(개인 계정 · 사업장 가입 요청 · 신규 사업장 등록)를 검증할 대기 상태 더미데이터 확정

---

### 1. 대상 계정

세 플로우 모두 기존 `seed.py`에 이미 각 1건씩 존재 — 신규 생성 없이 자격증명과 대응 화면을 확정하는 작업

| 플로우 | 데이터 | 검토 화면 |
|---|---|---|
| 개인 계정 승인 | newbie@bakery.kr (윤서준), is_active=0, 소속 사업장 없음 | 개발자 콘솔 › 개인 계정 승인 |
| 사업장 가입 요청 | outside@otherco.kr (한소미) → 행복한 베이커리(id=4)에 pending 요청 | 가입 승인 관리(사업장 admin) › 가입 승인 관리 |
| 신규 사업장 등록 | dalkomm@example.com(달콤제과점) status=pending | 개발자 콘솔 › 신규 사업장 승인 |

---

### 2. 검증

| 항목 | 결과 |
|---|---|
| 개발자 계정으로 `GET /api/auth/users/pending` | ✅ newbie@bakery.kr 확인 |
| 개발자 계정으로 `GET /api/pending-registration/list` | ✅ dalkomm@example.com(달콤제과점) 확인 |
| 행복한 베이커리 admin(kim@bookkeep.ai)으로 `GET /api/business/join-requests` | ✅ 한소미 요청 확인 |

세 화면 모두 v11.1 · v11.2에서 재구성한 권한 경계(플랫폼 관리자 전용 vs 사업장 admin 전용)를 그대로 통과하는지까지 함께 확인됨
