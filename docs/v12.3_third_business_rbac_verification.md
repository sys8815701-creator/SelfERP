### 2. 검증

| 항목 | 결과 |

| 3계정 로그인 응답의 role | ✅ admin · accountant · employee 각각 정확히 반환 |
| 신규 사업장의 `GET /api/settings/role-access` | ✅ `null` — 커스텀 설정 없이 v12.1 · v12.2에서 수정한 코드 기본값으로 폴백 확인 |
