# v12.4 구현 보고서 — 소셜 로그인 리다이렉트 URI 포트 불일치 수정

**작성일**: 2026-07-21  
**범위**: 카카오 · 네이버 · 구글 소셜 로그인이 전부 실패하던 문제 원인 확인 및 수정

---

### 1. 증상

로그인 화면에서 카카오 · 네이버 · 구글 로그인 버튼을 눌렀을 때 세 경우 모두 콜백 단계에서 실패

---

### 2. 원인

`backend/.env`의 `KAKAO_REDIRECT_URI` · `NAVER_REDIRECT_URI` · `GOOGLE_REDIRECT_URI`가 전부 `http://localhost:8000/...`으로 설정되어 있었으나, 실제 백엔드는 `8002` 포트에서 실행 중(프론트엔드의 `NEXT_PUBLIC_API_URL`도 8002를 가리킴). 각 제공자가 로그인 승인 후 브라우저를 8000 포트로 돌려보내지만 그 포트에는 아무 서버도 없어 연결 실패로 이어짐 — 세 제공자 모두 같은 원인으로 동일하게 실패

---

### 3. 조치

- `backend/.env`의 3개 REDIRECT_URI를 전부 `http://localhost:8002/...`로 수정
- `.env` 변경은 `uvicorn --reload`의 파일 변경 감지 대상이 아니므로 백엔드 프로세스를 수동으로 재시작해 반영
- `GET /api/auth/{kakao,naver,google}/login` 3개 엔드포인트가 생성하는 인가 URL의 `redirect_uri` 파라미터가 전부 8002로 바뀐 것을 재확인

---

### 4. 남은 작업(수동, 외부 콘솔)

각 제공자의 개발자 콘솔에 등록된 리다이렉트 URI가 여전히 8000으로 남아있다면 제공자 쪽에서 "등록된 주소 불일치"로 거부하므로, 아래 항목은 계정 소유자가 직접 갱신 필요

| 제공자 | 콘솔 | 등록해야 할 값 |
|---|---|---|
| 카카오 | developers.kakao.com → 내 애플리케이션 → 카카오 로그인 | `http://localhost:8002/api/auth/kakao/callback` |
| 네이버 | developers.naver.com → Application 등록 정보 | `http://localhost:8002/api/auth/naver/callback` |
| 구글 | console.cloud.google.com → API 및 서비스 → 사용자 인증 정보 | `http://localhost:8002/api/auth/google/callback` |

---

| 항목 | 결과 |
|---|---|
| 백엔드 재시작 후 `/docs` 응답 | ✅ 200 |
| `GET /api/auth/kakao/login` 리다이렉트의 `redirect_uri` | ✅ 8002로 변경 확인 |
| `GET /api/auth/naver/login` 리다이렉트의 `redirect_uri` | ✅ 8002로 변경 확인 |
| `GET /api/auth/google/login` 리다이렉트의 `redirect_uri` | ✅ 8002로 변경 확인 |
