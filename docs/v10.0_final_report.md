## 4. 데이터 모델 현황

| 모델 수 | 테이블 수 |
|---------|----------|
| 30+ | 30+ |

주요 관계:
- `Business` → 모든 도메인 모델의 root (business_id FK)
- `Item` ← `BOMLine` ← `BOM` ← `ProductionOrder` ← `ProductionResult`
- `Item` ← `InventoryLog`
- `SalesOrder` ← `SalesOrderItem` / `Delivery` ← `DeliveryReturn`
- `Employee` ← `Payroll`, `Leave`, `Contract`

## 5. 알려진 제한사항 및 향후 개선 과제

| 항목 | 현황 | 향후 개선 |
|------|------|-----------|
| 권한 관리 | UI 레벨만 구현 | 서버 수준 JWT claims + RBAC 미들웨어 |
| 회계 장부 CSV | Journal에 business_id 없어 전체 조회 | Journal 테이블 마이그레이션으로 사업장 필터 추가 |
| 경로 최적화 | 배송지 앞 3자 단순 그룹핑 | 주소 파싱 라이브러리 또는 지도 API 연동 |
| 실시간 배송 추적 | 수동 상태 변경 | GPS/외부 배송 API 연동 |
| 급여 소득세 자동 계산 | 수동 입력 | 간이세액표 기반 자동 계산 |
| 테스트 커버리지 | 없음 | pytest + Jest 단위/통합 테스트 추가 |

```

├── backend/
│   ├── main.py                    ← 전체 라우터 등록
```

## 7. 결론

SelfERP v1.0~v9.4를 통해 소상공인 대상 풀스택 ERP 시스템이 완성되었다.

- **백엔드**: FastAPI + SQLAlchemy + MySQL, 40+ REST 엔드포인트
- **프론트엔드**: Next.js 15 App Router, 34개 대시보드 페이지
- **인증**: JWT + X-Business-Id 헤더 기반 다사업장 지원
- **자동화**: 재고 자동 조정, 배송 완료 → 수주 상태 연동, 반품 → 재고 복원

Railway 배포 준비 완료 (환경변수·MySQL 플러그인 설정만 필요).
