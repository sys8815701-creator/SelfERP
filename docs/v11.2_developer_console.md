# v11.2 구현 보고서 — 개발자(플랫폼 관리자) 콘솔

**작성일**: 2026-07-21  
**범위**: 사업장 소속과 무관한 플랫폼 운영자 계정 유형과 전용 콘솔 신설

---

### 1. 배경

v11.1에서 개인 계정 승인 · 신규 사업장 승인을 `is_platform_admin` 전용으로 이관하면서, 기존 `integrated/pending/page.tsx`의 "개인 계정 승인" · "사업자 등록 승인" 탭이 사업장 admin에게 403을 반환하게 됨 — 이 두 탭을 신설 콘솔로 이관하고 해당 페이지에서 제거

---

### 2. 백엔드

| 항목 | 내용 |
|---|---|
| `routers/platform.py` 신설 | `GET /api/platform/businesses`(전체 사업장 요약: 이름 · 사업자번호 · 대표자 · 소유자 이메일 · 구성원 수 · PRO 여부) · `PATCH /api/platform/businesses/{id}/pro`(PRO 토글) |
| 게이트 | 전부 `require_platform_admin`(`current_user.is_platform_admin`) |
| `schemas/user.py` | `UserResponse`에 `is_platform_admin` 필드 추가 — 로그인 · `/me` 응답에 포함 |
| `create_developer.py` 신설 | 플랫폼 관리자 계정 1회 생성 스크립트, 기존 데이터 삭제 없이 순수 추가만 수행 |

---

### 3. 프론트엔드

| 항목 | 내용 |
|---|---|
| `hooks/useRole.ts` | `useIsPlatformAdmin()` 추가 |
| `dashboard/developer/page.tsx` 신설 | 탭 3개 — 전체 사업장 · 개인 계정 승인 · 신규 사업장 승인, `is_platform_admin` 아니면 접근 거부 |
| `dashboard/layout.tsx` | 사이드바에 "플랫폼 운영" 섹션 추가(`is_platform_admin`일 때만 노출) · 사업장 0개 시 강제 리다이렉트에 플랫폼 관리자 예외 추가 |
| `integrated/pending/page.tsx` | "개인 계정 승인" · "사업자 등록 승인" 탭 제거, "가입 승인 관리"(사업장 가입 요청)만 잔존 |
